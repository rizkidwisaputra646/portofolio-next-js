## Overview
This project involved re-architecting a legacy monolithic e-commerce platform into a modern microservices-based system to support a 5x increase in traffic.

## The Problem
The existing system suffered from long deployment cycles, frequent downtime during peak sales, and difficulty scaling specific components like the search engine and checkout process.

## Architecture
We chose a microservices approach using **Go** for performance-critical services and **Node.js** for the frontend-facing API gateway.

### Key Components:
- **Product Service:** Managed catalog and inventory with Redis caching.
- **Order Service:** Handled transactions with strict ACID compliance.
- **Search Service:** Powered by Elasticsearch for real-time filtering.

## Results
- **99.99% Uptime** during Black Friday.
- **40% reduction** in page load times.
- Deployment frequency increased from bi-weekly to multiple times a day.

## Lessons Learned
Trade-offs are everywhere. Distributed systems bring complexity in observability and data consistency which we addressed using OpenTelemetry and event-sourcing patterns.
